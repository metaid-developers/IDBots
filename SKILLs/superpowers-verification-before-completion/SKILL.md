# Superpowers Verification Before Completion

Do not claim success without fresh evidence.

## Gate

Before saying the work is done, fixed, or passing:

1. identify the command that proves the claim,
2. run the full command now,
3. read the output and exit status,
4. compare the result to the claim,
5. only then state the outcome.

## Applies To

- tests,
- builds,
- bugfix claims,
- migration results,
- readiness to merge,
- readiness to hand back to the user.

## Not Good Enough

- "should pass now",
- "looks correct",
- "the code change is straightforward",
- "the earlier run passed",
- "the delegated worker said it was done".

Evidence first, claims second.
